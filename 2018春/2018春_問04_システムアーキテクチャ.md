# 2018年春期（平成30年度）応用情報技術者試験 午後 問4（選択）
## システムアーキテクチャ：クラウドサービス（C社／D社）

---

## 問題文

**問4** クラウドサービスに関する次の記述を読んで、設問1〜4に答えよ。

保険代理店であるC社は、同業の小規模な代理店数社を吸収合併することになった。現在、C社の業務システムは、販売管理、顧客管理、事務支援、保険計上の四つのサブシステムから構成されている。各サブシステムは、ベンダに委託して開発したアプリケーションや市販ソフトウェアを利用しており、その運用については、システム運用を請け負うD社に外部委託している。合併後の業務システムは、C社の業務システムを継続して使用する前提であるが、D社のクラウドサービスの利用を検討することになった。システム部のE部長は、部下のF課長に、費用の削減を目的としたクラウドサービスへの移行案を、既存の業務システムの使い勝手や可用性、セキュリティレベルを維持して策定するように指示した。

---

### 〔クラウドサービス移行案の策定〕

F課長は、D社の営業部長からクラウドサービスの内容について説明を受けた。クラウドサービスは、SaaS、PaaS、IaaSの三つのサービス形態があり、それぞれのサービス形態に、パブリッククラウドとプライベートクラウドの二つの提供形態がある。そこで、F課長は既存の業務システムをクラウドサービスに移行する場合、各サブシステムに対して、D社のクラウドサービスの最も適切な利用形態（サービス形態と提供形態の組合せ）を検討し、その結果をクラウドサービス移行案とすることにした。

### 〔仕様の変更〕

F課長がサブシステムごとにクラウドサービスの利用形態を選定した後に、合併後の営業社員の給与制度を業績連動型に変更することになった。そのため、販売管理サブシステムで管理されている営業社員の契約状況を利用して給与を計算するというシステム仕様の変更が必要となり、F課長は、D社が提供している市販ソフトウェアをカスタマイズして使用することにした。

F課長は、給与計算の仕様を変更したクラウドサービスの利用形態の選定結果を、E部長に提出した。ここで、複数の利用形態が選択可能であったので、費用が安価な利用形態を選択した。

## 解答と解説

### 設問1

**正解：イ、エ**

- イ：表1で、他のシステム・サブシステムとのデータ連携のうち「D社クラウドセンタ外」とのデータ連携が可能なのはIaaSの「自社で調達する。」の欄のみであり、PaaS・SaaSでは利用できない。したがって、市販ソフトウェアがD社提供OSで稼働してもクラウドセンタ外とのデータ連携が必要ならIaaSを選ぶ必要がある。→適切
- エ：表1「業務システム」の「自社開発アプリケーション」はIaaS・PaaSで自社調達可能であり、カスタマイズした市販ソフトウェアもアプリケーションとして自社調達すればIaaS・PaaSで利用できる。→適切
- ア：不適切（OS要件についての記述は本文にない）
- ウ：表1より、データ連携がPaaSで利用できるのはプライベートクラウドのみで、パブリッククラウドのPaaSでは利用できないため不適切。
- オ：パブリッククラウドの契約形態についての記述は本文にない。

**IPA公式：イ、エ**

保険計上サブシステムは、表3より「保険会社のシステム・販売管理と接続」（→データ連携○）、「稼働の変動：有」（→稼働の変動への対応○）、「ベンダに委託して開発したアプリケーションを利用」（→独自の業務への対応○、共通の業務への対応×）。

### 設問3

**正解：e = プライベートクラウド、f = IaaS**

- e：販売管理サブシステムは、他サブシステム（保険計上・顧客管理）とのデータ連携が必要であり、表1よりPaaSでデータ連携が利用できるのはプライベートクラウドのみ。
- f：顧客管理サブシステムは、D社が提供するOS以外のOSを利用しており、表1の「業務システム」欄より、自社開発アプリケーションを自社調達できるのはIaaS・PaaSだが、D社提供OS以外を使う（OSも自社調達が必要）にはIaaSを選ぶ必要がある。

**IPA公式：e = プライベートクラウド、f = IaaS**

### 設問4

**(1) 正解：サブシステムの名称＝事務支援サブシステム、サービス形態＝PaaS、提供形態＝プライベートクラウド**

給与計算のために販売管理サブシステムの契約状況データを利用する必要が生じたため、当初SaaSのパブリッククラウドを予定していた事務支援サブシステムは、市販ソフトウェアをカスタマイズして利用する必要が生じ、SaaS（カスタマイズ不可）が使えなくなった。カスタマイズしたソフトウェアが利用でき、かつ販売管理サブシステムとのデータ連携が可能な形態として、PaaSのプライベートクラウドを選択した。

**(2) 正解例：販売管理サブシステムとのデータ連携が必要だから（30字以内）**

給与計算のために、販売管理サブシステムで管理する営業社員の契約状況データとのデータ連携が必要になったため、データ連携可能な利用形態を選ぶ必要があった。

**IPA公式：**
(1) サブシステムの名称＝事務支援サブシステム、サービス形態＝PaaS、提供形態＝プライベートクラウド
(2) 販売管理サブシステムとのデータ連携が必要だから

## 参考：主要キーワード

| 用語 | 説明 |
|------|------|
| SaaS / PaaS / IaaS | クラウドサービスの提供範囲の違いによる分類。SaaSはアプリケーションまで、PaaSはミドルウェア・OSまで自社管理不要、IaaSはハードウェア・ネットワークのみ提供され、それ以外は自社で構築・調達する |
| パブリッククラウド / プライベートクラウド | パブリッククラウドは複数企業でリソースを共有し安価だが処理状況の影響を受ける。プライベートクラウドは自社専有でリソースを自由に配分できるが高価 |
| データ連携（クラウドセンタ内／外） | クラウドセンタ内のシステム間データ連携とクラウドセンタ外（自社や取引先）とのデータ連携で、利用可能なサービス形態・提供形態の制約が異なる |
| ベストエフォート型サービス | 提供される性能や品質について保証をせず、可能な範囲で最善を尽くす形で提供されるサービス方式。パブリッククラウドの典型的な特徴 |
| システム要件の対応表 | 各サブシステムが必要とする要件（データ連携、稼働変動対応、共通/独自業務対応）を整理し、最適なクラウド利用形態選定の判断材料とする手法 |
