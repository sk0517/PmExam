# 応用情報 午後 書き起こしMDの照合・修正スキル

このリポジトリは、応用情報技術者試験 午後問題（PDF）を視覚的に読み取って書き起こしたMarkdown（`YYYY季/YYYY季_問NN_分野.md`）を蓄積している。書き起こしには**捏造（実在しない文章の生成）・段落や図表の欠落・数値/記号の誤変換・解答誤り**が混入していることが分かっている。このスキルは、**元PDFのページ画像を正(ground truth)として既存MDを1問ずつ検証・修正する**手順を定める。

土台は `anthropic-skills:protected-exam-pdf-transcriber` の**修正モード**。その鉄則（画像が正・最小差分・判読不能は推測で埋めない・高リスク要素は逐語転写）をそのまま適用したうえで、本リポジトリ固有の手順を加える。

## 最重要の制約（セキュリティ／著作権）

PDFは `Encrypted: yes (copy:no)` のスキャン画像。**機械的テキスト抽出（pdftotext / pypdf / pdfimages / OCR）は技術的保護手段の回避に当たるため行わない。** 日本の著作権法では回避して行った複製は私的複製の例外外となり、個人学習でも侵害になり得る。**許可されるのはラスタライズ（ページを画像として描画）して視覚的に読むことだけ**。poppler CLI（pdftoppm）は環境に無い。**PyMuPDF (fitz) を 3x（≒300dpi）・PNG でラスタライズする。**

## ファイルとパスの規約

- 問題PDF: `pdfs/YYYY<和暦>h_ap_pm_qs.pdf`（春=h）/ `...a_ap_pm_qs.pdf`（秋=a）。公式解答: `..._ans.pdf`。
  - 例: 2015秋=`2015h27a`、2020秋=`2020r02o`、2021春=`2021r03h`。実ファイル名は `ls pdfs/` で確認する。
- MD: `YYYY季/YYYY季_問NN_分野.md`（季=春/秋）。パスは Glob `YYYY季/*問NN*` で特定する。
- **印刷ページ番号 = PDFページ index − 1**（表紙が index 0）。問題一覧は index 2 付近。
- 午後の構成: 問1=必須(情報セキュリティ)、問2〜11=選択(4問選択)、間に〔メモ用紙〕の空白ページ。

## 手順（1問ずつ）

### 1. ラスタライズ（その年で未生成なら1回だけ）

scratchpadに年ごとのディレクトリを作って全ページPNG化する。既に `<tag>_qs_pages/` があれば再生成不要。

```python
import fitz, os
base=r'C:/WorkSpace/code/PmExam/pdfs/'
for tag,fn in [('2015aki_qs_pages','2015h27a_ap_pm_qs.pdf'),
               ('2015aki_ans','2015h27a_ap_pm_ans.pdf')]:
    os.makedirs(tag,exist_ok=True)
    d=fitz.open(base+fn)
    for i,p in enumerate(d):
        p.get_pixmap(matrix=fitz.Matrix(3,3)).save(f'{tag}/page_{i:02d}.png')
```

scratchpad基点: `C:/Users/oto57/AppData/Local/Temp/claude/C--WorkSpace-code-PmExam/<session>/scratchpad/`。タグ命名例: `2015aki`(秋)/`2015haru`(春)。

### 2. ページ範囲の特定と読取

問題一覧ページ（index 2付近）で分野・テーマを確認 → 対象問のページ範囲を割り出す。該当ページ画像を Read で**視覚的に**開いて読む。1問はふつう本文数ページ＋設問。表・図・擬似コード・SQL・数値は特に注意して見る。判読困難なら該当ページだけ更に高dpiで再描画する。

### 3. MDとの突き合わせ（照合の観点）

既存MDを Read し、画像を正として次を確認する。**画像が明確に否定している箇所だけ**を直す（「ついでに読みやすく」はしない）。

- **捏造**: 画像に無い文章・箇条書き・表のセルをMDが書いていないか。最頻の事故。
- **欠落**: 段落・図・表・注記・空欄（`[ a ]`等）がMDから丸ごと落ちていないか。空欄は設問と対応するので特に重要。
- **誤変換**: 数値・単位・記号（○△×）・評価点・助詞重複・カタカナ長音・専門用語の取り違え。表セルと擬似コード/SQLは逐語で照合。
- **設問文**: 設問の字数制限・下線番号・選択肢（解答群）が画像と一致するか。
- **タイトル**: `（必須）`/`（選択）`の別（必須は問1のみ）。

### 3.5. 正誤表（errata）の優先適用 ★重要な例外

IPAは出題後に**正誤表**を出すことがある。その場合、**PDF本体の記載が「誤った情報」、正誤表の記載が「正しい情報」**であり、**この箇所だけは「PDF画像が正」の原則を上書きして、正誤表の正しい情報をMDに反映する**。

- 正誤表は `修正メモ/正誤表一覧.md` に集約している。**照合の前に必ずこのファイルを読み**、対象の年・問に該当項目があれば、PDF画像ではなく正誤表の「正しい情報」に合わせてMDを直す。
- 該当があれば、コミットメッセージに「正誤表反映」と明記する（通常の書き起こし崩壊修正と区別するため）。

### 4. 解答の照合

`<tag>_ans/` の公式解答画像を読み、MDの各設問の解答値・記号・計算結果・「IPA公式」ラベルが公式と一致するか確認する。相違があれば公式に合わせ、解説中の「IPA公式：」も官報の正確な表記に揃える。

### 5. 修正の適用と報告

- Editで最小差分修正。崩壊が広範な節（表の全面捏造・段落まるごと欠落）は、その節だけ画像どおりに書き直す。
- 判読不能は `[判読不能: 位置]` と明記（推測で埋めない）。
- 修正が無ければ「崩壊なし・修正不要」と明記する（**無修正も正当な結論**。むやみに書き換えない）。
- **1問ごとにコミット**。メッセージは日本語で、何をどう直したか（数値・記号の変更は必ず列挙）。末尾に:
  ```
  Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
  ```
- Windowsの `LF will be replaced by CRLF` 警告はエラーではない。無視してよい。

## 注意

- 全11問を一度に処理するとトークンを要する。ユーザの指示に従い、指定された問（または年）だけを対象にする。
- 修正メモ（`修正メモ/`）は過去の読取に基づくため、それ自体に誤りが残り得る。**最終的な正はあくまで再ラスタライズしたPDF画像**であり、メモは参考に留める。
