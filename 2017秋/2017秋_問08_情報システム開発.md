## 問題文

**問8** ソフトウェア適格性確認テストに関する次の記述を読んで、設問1〜4に答えよ。

W法人は技術者の国家資格認定試験を実施している団体である。グローバルに活躍できる技術者を育成するために、新たな技術者認定試験（以下、新試験という）を導入することが決まった。新試験は4種類の試験を組み合わせて合格者を決定する。そこで、4種類の試験の成績を基に合否を判定するシステム（以下、合否判定システムという）を開発して、そのシステムの動作を確認するためのテストを行うことにした。

---

### 〔新試験の実施方法〕

新試験では、次の4種類の試験を組み合わせる。

Ⅰ　英語（筆記試験）：得点は1点刻みで100点満点（以下、この筆記試験の得点をXとする）

Ⅱ　専門科目（筆記試験）：得点は1点刻みで100点満点（以下、この筆記試験の得点をYとする）

Ⅲ　英語（面接試験）：得点は5点刻みで100点満点（以下、この面接試験の得点をORAL_engとする）

Ⅳ　技術者適性（面接試験）：得点は1点刻みで1〜4点（以下、この面接試験の得点をORAL_tecとする）

新試験は次の2段階で行われる。

第1段階：筆記試験（Ⅰ英語 と Ⅱ専門科目）

第2段階：面接試験（Ⅲ英語 と Ⅳ技術者適性）

第1段階の判定基準を満たした受験者だけが第2段階に進み、第2段階の判定基準を満たした受験者が新試験の合格者となる。

### 〔3変数のドメイン分析〕

第2段階の判定基準（条件3、4）においてENGLISH、WRITTEN、ORAL_tecの3変数の境界値テストを行う。このように複数の変数の境界値が関係するテストケースの設定を見つけるために、Binderのドメイン分析を利用する。Binderのドメイン分析とは、ある変数の境界値についてテストを行うために、他の変数を有効同値の中の値とする方法である。それぞれのドメインは境界によって定義されるので、テストすべき値は、仕様で指定される境界上の値（onポイント）、及び境界の近傍にあって境界を挟んでonポイントに最も近い値（offポイント）となる。offポイントは、境界が閉じていれば（等号を含む不等式の場合）ドメイン外の値になり、境界が開いていれば（等号を含まない不等式の場合）ドメイン内の値となる。一つの変数の境界をチェックするときに、他の変数は真偽に影響を与えないよう境界上でないドメイン内部の値（inポイント）を選ぶ。

## 解答と解説

### 設問1

**正解：（100，100）、（60，100）、（60，70）、（100，30）**

条件1（X≧60）と条件2（WRITTEN＝X+Y≧130）を満たす領域は、X、Yがそれぞれ0〜100の範囲内で、X≧60かつX+Y≧130を満たす部分である。この領域の境界は、X＝60、X＝100（上限）、Y＝100（上限）、X+Y＝130の4直線で構成される四角形となる。

各頂点を求めると、右上の頂点は（X＝100、Y＝100）。そこから左回り（反時計回り）に、Y＝100とX＝60の交点（60，100）、X＝60とX+Y＝130の交点（60，70）、X+Y＝130とY下限（この場合Yの最小値でX上限に対応する点、X＝100のときY＝30）である（100，30）。したがって、頂点は**（100，100）、（60，100）、（60，70）、（100，30）**の順となる。

**IPA公式：（100，100），（60，100），（60，70），（100，30）**

---

### 設問2

**正解：a = 30、b = 250**

条件4を連立不等式で表す際、表1の境界（WRITTEN190以上でORAL_tec2以上が条件4を満たす、WRITTEN160以上190未満でORAL_tec3以上、WRITTEN130以上160未満でORAL_tec4以上）を1本の直線WRITTEN + m×ORAL_tec ≧ n で近似する。

境界点（WRITTEN=190, ORAL_tec=2）と（WRITTEN=160, ORAL_tec=3）を通る直線の傾きから、m＝(190-160)/(3-2)×(-1)？ 実際には2点(190,2)と(160,3)を通る直線WRITTEN = n - m×ORAL_tecを考えると、190 = n - 2m、160 = n - 3m。これを解くとm=30、n=250。もう1点(130,4)で確認：n - 4m = 250 - 120 = 130で一致する。したがってm＝**30**（a）、n＝**250**（b）。

**IPA公式：a=30、b=250**

### 設問3

**(1) 正解：ENGLISHの境界値チェック**

ケース1（ENGLISH on=140）とケース2（ENGLISH off=d）は、他の変数（ORAL_tec、WRITTENはinポイント）を固定した上でENGLISHの値だけを変化させているため、テストケースの目的は**ENGLISHの境界値チェック**である。

**IPA公式：ENGLISHの境界値チェック**

**(2) 正解：d = 141、e = 129**

ENGLISHの境界は条件3「ENGLISH＞140」であり、境界が開いている（等号を含まない）ため、offポイントは境界のすぐ外側でドメイン外となる値、すなわち140より小さい141に最も近いドメイン内の値ではなく、offポイントの定義（境界を挟んでonポイントに最も近い値）から、on=140（ドメイン外＝不合格側）に対し、off=**141**（ドメイン内側、合格側）となる。

WRITTENの境界は条件2「WRITTEN≧130」であり、境界が閉じている（等号を含む）ため、onポイントの130に対し、offポイントは境界の外側（ドメイン外＝130未満）に最も近い値、すなわちe＝**129**となる。

**IPA公式：d=141、e=129**

**(3) 正解：（ウ，160）、（オ，1）、（ケ，190）**

ケース4は「ORAL_tecの境界値チェック」であり、ORAL_tecの値だけを変化させ、他の変数（ENGLISH、WRITTEN）はinポイントで固定する必要がある。

ENGLISHの欄では、（ア）on＝140、（イ）off＝141は既に埋まっているように見えるが、実際にはinポイントを使うべき箇所（ウ）に値が未設定であり、他ケースと同じinポイントの**160**を設定する（ウ，160）。

ORAL_tecのoffポイント（オ）は、境界（ORAL_tec≧2）が閉じているため、境界外側に最も近い値、すなわち**1**を設定する（オ，1）。

WRITTENの欄では、inポイントの（ケ）に他ケースと同じ**190**を設定する（ケ，190）。

**IPA公式：（ウ，160），（オ，1），（ケ，190）**

### 設問4

**正解例：条件1が偽、条件2が真、条件5が偽となる場合が成立しないから**

複数条件網羅では、本来2^3＝8通りの真偽の組合せが必要となる（条件1、条件2、条件5各々が真偽の2通り）。しかし、条件1（X≧60）が偽（X<60）であるにもかかわらず条件2（WRITTEN＝X+Y≧130）が真になるためには、Y>70程度の高得点が必要だが、条件5（Y>50）が偽（Y≦50）という組合せは、X<60かつY≦50の場合WRITTEN=X+Y<110となり130以上になり得ず、論理的に成立しない。すなわち、**条件1が偽、条件2が真、条件5が偽となる場合が成立しないから**、本来8件必要な組合せのうち1件が実現不可能であり、7件で複数条件網羅率100%を達成できる。

**IPA公式：条件1が偽，条件2が真，条件5が偽となる場合が成立しないから**

## 参考：主要キーワード

| 用語 | 説明 |
|------|------|
| 境界値分析（Boundary Value Analysis） | 仕様で定められた境界（on）とその近傍（off）に着目してテストケースを設計する技法。境界が閉区間か開区間かでoffポイントの位置が変わる |
| Binderのドメイン分析 | 複数変数が関わる判定条件に対して、ある変数の境界をテストする際、他の変数を境界に影響しないinポイントに固定してテストケースを設計する手法 |
| 複数条件網羅（Multiple Condition Coverage） | 判定条件を構成する各条件の真偽の全組合せを網羅するテスト基準。条件数がnの場合最大2^n通りの組合せが必要 |
| onポイント／offポイント／inポイント | on：境界上の値、off：境界を挟んでonに最も近い値（境界の開閉で内外が変わる）、in：境界に影響しないドメイン内部の値 |
| 論理的に成立しない組合せ | 複数の条件が数式的に連動している場合、ある条件の組合せが数学的に実現不可能となることがあり、その分テストケース数を削減できる |
