## 解答と解説

### 設問1

**(1) 正解：FWf**

L3SWの経路表には、Webサイトb DMZへの経路と①デフォルトルートが登録されている。F社内からインターネット方向へのデフォルトルートのネクストホップはFWf（F社のファイアウォール）となる。

**(2) 正解：L3SW、FWb、L2SWb**

運用PCから（広域イーサ網経由で）DNSサーバb（b-DC DMZ内）にアクセスする経路：  
運用PC → L3SW → FWf（←経由なし？） → 広域イーサ網 → FWb → L2SWb → DNSサーバb  
ただし問題の②設定（デフォルトルート設定）の下での経由機器として、F社LAN内の機器のみを問うていると解釈すると L3SW, FWb, L2SWb となる。

---

### 設問2

**(1) 正解：a = miap.example.jp**

顧客がアクセスするURLはグローバルIPアドレスに紐付いたFQDNであるmiap.example.jp。表1の項番2に記載されているWebAPサーバbのグローバル向けFQDN。

**(2) 正解：apb.f-sha.example.lan**

SSHによるログインはF社内LAN（広域イーサ網）経由のプライベートアドレスを使用する。表1の項番4に登録されているWebAPサーバbのLAN向けFQDN「apb.f-sha.example.lan」を使用してログインする。

---

### 設問3

**(1) 正解**

| 空欄 | 正解 | 解説 |
|------|------|------|
| **b** | DNSサーバc | 運用PCのDNS設定にセカンダリDNSとしてDNSサーバcを登録する |
| **c** | ゾーン | プライマリからセカンダリへのゾーン転送設定を行う |
| **d** | TTL | メンテナンス前にTTLを短くしておくことでAレコード削除後にキャッシュが早く無効化される |

**b = DNSサーバc**：プライマリDNSサーバbに加え、代替（セカンダリ）としてDNSサーバcを運用PCに登録することで、片系障害時も名前解決可能になる。

**c = ゾーン**：DNSのゾーン転送（Zone Transfer）設定。プライマリのゾーン情報をセカンダリに複製（AXFR/IXFR）する仕組み。

**d = TTL**：TTL（Time To Live）を小さくすることで、DNSキャッシュの保持時間が短縮され、Aレコード削除後に短時間でキャッシュが無効化される。これにより手順(iii)の待ち時間を短縮できる。

**(2) 正解：Mシステムの応答速度が低下することがある。（23字）**

メンテナンス中は片系（1サイト）のみでサービス提供するため、負荷が集中しWebAPサーバの応答速度が低下するリスクがある。

**(3) 正解：ログイン中の利用者がいないこと（17字）**

MシステムはWebサイトへのログインが必要なシステム。メンテナンス対象WebAPサーバにログイン中の利用者が残っている状態で作業を開始すると、その利用者のセッションが切断されてしまう。そのため、ログイン中の利用者がいないことを確認してから作業を開始する。

## 参考：主要キーワード

| 用語 | 説明 |
|------|------|
| DNSラウンドロビン | 同一のFQDNに複数のIPアドレスを登録し、順番に返すことで負荷分散する方式 |
| FQDN（Fully Qualified Domain Name） | ホスト名+ドメイン名を含む完全修飾ドメイン名 |
| Aレコード | DNSリソースレコードの種類。FQDNとIPv4アドレスを対応付ける |
| TTL（Time To Live） | DNSキャッシュの有効期間（秒）。小さいほどキャッシュが早く更新される |
| NAT（Network Address Translation） | プライベートIPとグローバルIPを相互変換する技術 |
| プライマリDNS / セカンダリDNS | プライマリが正式なゾーン情報を持ち、セカンダリはゾーン転送で複製を保持 |
| ゾーン転送（Zone Transfer） | プライマリDNSからセカンダリDNSへゾーン情報を複製する仕組み（AXFR/IXFR） |
| DMZ（DeMilitarized Zone） | ファイアウォールで保護された中間ネットワーク。Webサーバ等を配置 |
| L3スイッチ | レイヤー3（ネットワーク層）でルーティングを行うスイッチ |
| スケールアウト | サーバ台数を増やして処理能力を拡張する水平スケーリング手法 |
