## A. 重大（解答の誤り）

### A-1. 設問1のbの解答が誤り（140・144行目）
- 現状: 「b=キャンペーン商品（エンティティ名）」

- 空欄bは図1の**商品エンティティとキャンペーン商品エンティティの間の関連**を問うもの（エンティティ名ではない）。商品1件は複数のキャンペーン商品に対応するため「商品→キャンペーン商品」の1対多の矢印が正解。
- 解説も全面書き直し: 一つの商品は複数のキャンペーンの対象になり得る（キャンペーン商品はキャンペーン番号+キャンペーン商品番号を主キーとする）ため、商品からキャンペーン商品への1対多（→）となる。

## B. 重大（SQL・図の崩れ）

### B-1. 図2のSQL文が原本と大きく異なる（62〜77行目）
PDF原本の正しいSQL（空欄c・d・eの位置に注意）:

```sql
SELECT CK.キャンペーン種別, COALESCE(UK.売上計, 0) AS 売上計,
  COALESCE(UK.原価計, 0) AS 原価計,
  COALESCE(UK.売上計, 0) - COALESCE(UK.原価計, 0) AS 粗利計,
  CK.キャンペーン実施金額計
[　c　]
(SELECT キャンペーン種別, SUM(キャンペーン実施金額) AS キャンペーン実施金額計
 FROM キャンペーン実施
 WHERE キャンペーン番号 = :キャンペーン番号
 GROUP BY キャンペーン種別) CK
[　d　]
(SELECT U.キャンペーン種別, SUM(M.商品個数 * M.商品単価) AS 売上計,
  [　e　] AS 原価計
 FROM 売上 U
  INNER JOIN 売上明細 M ON U.売上番号 = M.売上番号
  INNER JOIN 商品 S ON M.商品番号 = S.商品番号
 WHERE U.キャンペーン番号 = :キャンペーン番号
 GROUP BY U.キャンペーン種別) UK
ON CK.キャンペーン種別 = UK.キャンペーン種別
```

現行MDとの主な相違:
1. SELECT句: `COALESCE(UK.売上計,0)`（UK側）が正。MDは `CK.売上計` と誤記。**`CK.キャンペーン実施金額計` の列が欠落**（集計対象の「実施金額」）
2. 空欄cは**SELECT句の直後**（=FROM）に位置する。MDでは1つ目のサブクエリの中に `FROM キャンペーン実施` が書かれ、空欄の位置関係が崩れている
3. 1つ目のサブクエリのGROUP BYは「**キャンペーン種別**」のみ（MDは「キャンペーン番号, キャンペーン種別」と誤記）
4. 2つ目のサブクエリのSELECT句に「**SUM(M.商品個数 * M.商品単価) AS 売上計**」がある（MDでは欠落。UK.売上計の由来）
5. MDのSQL中に `[　b　]` が混入している（bは図1のE-R図の空欄。SQLには登場しない）
