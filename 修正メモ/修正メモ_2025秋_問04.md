# 修正メモ：2025秋_問04_システムアーキテクチャ.md

照合元：`pdfs/2025r07a_ap_pm_qs.pdf` p.24〜28（問4本体）、公式解答 `pdfs/2025r07a_ap_pm_ans.pdf`
※解答は全問公式と一致。ただし**電力量の測定値が捏造**されており（合計6,510は偶然一致）、設問4(1)の計算根拠が完全に誤っている。通信帯域の説明段落（設問3の根拠）も欠落。

## A. 重大（解答の根拠・設問に影響）

1. **電力量の測定値が捏造（行39）— 設問4(1)の計算根拠**
   - 正：「Q君は、ボードコンピュータの動作に必要な1時間当たりの電力量を測定した。その結果、**何も処理をしていない起動状態で1,500mWh、これに加えて**各装置の使用率が100%の場合、CPUは2,000mWh、GPUは3,000mWh、動画を圧縮・伸長する専用チップは400mWh、**モバイル通信装置は500mWh、カメラモジュールは100mWh**であった。また、電力量は装置の使用率に比例することが分かった。」
   - MDは起動状態1,500mWhが欠落し、「カメラモジュール1,800mWh」「モバイル通信装置100mWh」と値を捏造。
   - 設問4(1)の解説表（行190〜199）も全面書き直し：
     | 項目 | 100%時 | 使用率 | 電力量 |
     |---|---|---|---|
     | 起動状態（ベース） | 1,500 | − | 1,500 |
     | CPU | 2,000 | 80% | 1,600 |
     | GPU | 3,000 | 90% | 2,700 |
     | 専用チップ | 400 | 90% | 360 |
     | モバイル通信装置 | 500 | 50% | 250 |
     | カメラモジュール | 100 | 100% | 100 |
     | **合計** | | | **6,510 mWh** |

2. **通信帯域の説明段落が欠落（行45）— 設問3の根拠**
   - 正：「…動画を無圧縮で**伝送**する場合、`[c]` **Gビット／秒（以下、bpsという）**の通信帯域が必要となる。**しかし、観覧席で安定的に利用できる4Gのモバイル通信の上り側の実効速度は数十Mbps程度であり、通信帯域が不足する。また、4K解像度の動画の一部を切り取ってFHD解像度（1,920×1,080ピクセル）の動画として伝送する場合にも通信帯域は十分ではない。**」
   - 設問3の解答「モバイル通信の帯域が不足する制約」の直接根拠。MDでは丸ごと欠落。解説（行182）の「4Gで約400Mbps程度」も誤りで「**数十Mbps程度**」に修正。

3. **下線④の文言誤り（行83）— 設問4(2)の対象**
   - 正：「この問題の原因を調査したところ、**ボール位置の検出処理で利用するGPUの使用率が高かったので**、GPUが処理するデータ量を**低減する対応**を検討した。**データ量を低減する複数の方法のうち、広いグラウンドから小さなボールを検出する必要がある点、ボールが高速に移動する点を考慮し**、ボール位置の検出処理の前に<u>④**撮影した**動画のデータ量を小さくする処理を追加</u>した。**この結果、追加した処理に必要な電力量と比較し、GPUの使用率の低減によって削減される電力量の方が多く**、動画を試合終了まで配信できるようになった。」
   - MDの「④集積した動画」「動画のデータ量と、GPUの使用率の低減によって必要な電力量が低減する方が多く」は崩壊。

4. **空欄d・e前後の文（行47）**
   - 正：「このことから、モバイル通信を用いて動画を配信するためには、**動画の圧縮などの対応策**が必要である。代表的な動画の圧縮方式には `[d]` があり、**専用チップである `[e]` を使用して動画を圧縮する。専用チップである `[e]` を利用することには、**<u>①ソフトウェアで圧縮する場合と比較してメリットがある</u>。」（空欄eは2回出現する）
