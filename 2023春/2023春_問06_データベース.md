### 〔達成状況集計リストの作成〕

H主任は、各組織がKPI達成状況を評価するための情報として、毎月末に達成状況集計リスト（以下、集計リストという）を作成し、提示することにした。

集計リスト作成は、オンライン停止時間帯の日次バッチ処理終了後の月次バッチ処理によって、処理結果を一時表に出力して後続処理に連携する方式で行うことにした。

集計リスト作成処理の概要を表4に示す。

### 表4 集計リスト作成処理の概要

> | 項番 | 入力表 | 出力表 | 集計日における処理内容 |
> |------|--------|--------|--------------------------|
> | 1 | 所属、役職 | 従業員_所属_一時 | 一般職従業員と所属組織の対応表を作成する。 |
> | 2 | 月別個人目標 | 従業員ごと_目標集計_一時 | 年度開始年月から集計月までの従業員、KPI項目ごとの目標個人集計値を求める。 |

集計リスト作成処理のSQL文を図2に示す。ここで、TO_DATE関数は、指定された年月日をDATE型に変換するユーザー定義関数である。関数COALESCE(A,B)は、AがNULLでないときはAを、AがNULLのときはBを返す。また、":年度開始年月日"、":年度開始年月"、":集計年月日"、":集計年月"は、該当の値を格納する埋込み変数である。

H主任は、図2の項番4のSQL文の設計の際に、次に示す考慮を行った。

- 表2の評価方法に従い、管理職の従業員データは対象に含めず、年度途中入社と、年度途中退職の従業員データについては出力しないように、抽出日に退職している従業員データを出力しない"従業員_所属_一時表"と、年度開始時点で入社していない従業員データを出力しない"従業員ごと_目標集計_一時表"を `[　c　]` によって結合しておく。
- `[　c　]` による結合結果と、実績がある場合だけレコードの存在する"従業員ごと_実績集計_一時表"を `[　d　]` によって結合しておく。また、①**実績個人集計がNULLの際は、0を設定しておく**。

## 解答と解説

### 設問1

**正解：a ＝ 1対多（組織 → 所属）、b ＝ 従業員コード**

- **a**：組織エンティティと所属エンティティの間の関連。一つの組織には複数の従業員の所属が対応し、一つの所属は一つの組織に対応するので「1対多」（組織 → 所属）となる。
- **b**：従業員エンティティの主キーとなる属性。所属・日別個人実績・月別個人目標から参照される「従業員コード」が入る（実線の下線＝主キー）。

---

### 設問2

**(1) 正解**

| 空欄 | 正解 | 解説 |
|------|------|------|
| **c** | INNER JOIN（内部結合） | 従業員_所属_一時と従業員ごと_目標集計_一時を結合。両表に存在する（＝一般職かつ年度開始時に入社済みで目標がある）従業員だけを対象にするため内部結合 |
| **d** | LEFT OUTER JOIN（左外部結合） | cの結合結果と従業員ごと_実績集計_一時を結合。実績のない従業員も残す必要があるため左外部結合。実績側はNULLになる |
| **e** | BETWEEN | 「（値）BETWEEN 下限 AND 上限」で期間・範囲を判定する |
| **f** | B.職務区分 = '02' | 一般職（職務区分'02'）の従業員だけを対象にする条件 |
| **g** | GROUP BY 従業員コード, KPIコード | 従業員・KPI項目ごとにSUMで集計するためのグループ化 |
| **h** | 組織ごと_目標実績集計_一時 | 項番4の出力先の一時表（項番5でも参照される） |
| **i** | COUNT(*) | 組織・KPI項目ごとの対象従業員数を数える集合関数 |

**各空欄の補足：**

- **c＝INNER JOIN**：抽出日に退職している従業員を出力しない従業員_所属_一時表と、年度開始時点で入社していない従業員を出力しない従業員ごと_目標集計_一時表の双方に存在する従業員に限定するため、内部結合を用いる。
- **d＝LEFT OUTER JOIN**：実績がある場合だけレコードが存在する従業員ごと_実績集計_一時表を結合しても、実績のない従業員を脱落させないよう、cの結合結果を左表とする左外部結合を用いる。実績個人集計がNULLになった場合はCOALESCEで0を設定する（下線①）。
- **e＝BETWEEN**：項番1・2・3のいずれも、日付・年月が指定範囲内かどうかを判定している。
- **f＝B.職務区分 = '02'**：役職表を結合し、職務区分が一般職（'02'）の従業員だけを対応表に載せる。
- **g＝GROUP BY 従業員コード, KPIコード**：SUM集計のため、従業員・KPI項目単位でグループ化する。
- **h＝組織ごと_目標実績集計_一時**：項番4で作成し、項番5で集計リストの基データとして参照する一時表。
- **i＝COUNT(*)**：目標平均値・実績平均値を求める際の分母となる対象従業員数を数える。

**(2) 正解：あるKPI項目について、月別個人目標のデータはあるが、日別個人実績のデータが1件も存在しない場合（40字以内）**

項番4では、目標集計（従業員ごと_目標集計_一時）を左表、実績集計（従業員ごと_実績集計_一時）を右表として左外部結合している。目標のデータは存在するが、年度開始年月日から集計年月日までの間に該当KPI項目の日別個人実績が一度も登録されなかった従業員では、右表にレコードがなく実績個人集計がNULLになる。この場合にCOALESCEで0を設定する（下線①）。

## 参考：主要キーワード

| 用語 | 説明 |
|------|------|
| KPI（Key Performance Indicator） | 重要業績評価指標。目標達成度を測るための定量的な指標 |
| INNER JOIN | 内部結合。両方のテーブルに一致するレコードのみを返す |
| LEFT OUTER JOIN | 左外部結合。左テーブルの全レコードを残し、右テーブルに一致しない場合はNULLを返す |
| BETWEEN A AND B | A以上B以下の範囲条件を指定するSQL演算子 |
| GROUP BY | 指定した列でグループ化し、集合関数（SUM/COUNT等）を適用する |
| COUNT(*) | テーブルの行数を数える集合関数。NULL行も含む |
| SUM() | 数値列の合計を返す集合関数 |
| COALESCE(A, B) | AがNULLの場合にBを返す関数。NULLを0などに置換する際に使用 |
| 一時表 | 処理の中間結果を格納する一時的なテーブル。バッチ処理で多用 |
| E-R図（Entity-Relationship Diagram） | エンティティ（表）とそのリレーションシップ（関連）を表す設計図 |
